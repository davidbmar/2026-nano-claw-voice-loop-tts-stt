IMPLEMENTATION SPEC — WHAT THE MVP WIRES UP

1 · Cost ledger (both repos)

Every call already leaves receipts: nano-claw's phone metrics DB (phone_calls table: call id, duration, turns) and riff's per-session artifacts. The MVP adds one table — cost_ledger(call_id, business, flow, component, units, unit_kind, usd) — written at call end from data the pipeline already has: Telnyx duration, STT seconds, per-turn token counts the providers return, TTS characters. No new instrumentation in the hot path.

2 · Pricing table (shared, versioned)

One pricing.json per repo (synced from the scheduler-model-costs skill, which already tracks $/1M rates and refresh dates). Ledger rows store units; dollars are computed at read time — so a price change re-prices history correctly.

3 · /api/costs endpoint

Aggregates the ledger: totals, by-component, by-business, by-flow, by-customer (caller-number hash → unique-customer counts, calls per customer, min/median/max minutes). The what-if scenarios (model swap, 24×7 extrapolation) are pure arithmetic over the same aggregates — exactly what this page's toggle does with static data.

4 · This page, served live

This mockup becomes the console page: nano-claw serves it at /costs; riff's version adds the multi-business registry it already has (phone-lines.yaml). Same design tokens as the voice console.

Illustrative data: model pricing & latency from the 2026-07 supervisor bake-off (11-scenario paid eval, results archived in tools/goal_region_eval/); business stats are sample shapes for design review. Telephony at $0.007/min + $1.00/DID·mo. Realistic duty = 13.9% of 24×7 (business-hours call load model from the scheduler-model-costs skill).
